## 📁 工作区边界

### ✅ 祖龙团队工作区（可操作）

```
/root/.openclaw/workspace-manager/          # 祖龙团队工作区
├── team/                                   # 团队管理文档
├── projects/                               # 团队项目（可以有多个）
│   ├── qingluan-org/                       # 青鸾组织架构系统
│   └── [未来其他项目]/                     # 团队未来开发的其他项目
├── memory/                                 # 团队记忆
└── reports/                                # 报告文档
```

**规则**：
- ✅ 可以操作 `workspace-manager/` 下的**所有目录**
- ✅ 可以开发新项目（只要在 `workspace-manager/` 下）
- ✅ 可以读取/修改/部署此工作区内的任何文件

---

### ❌ 外部工作区（禁止修改）

```
/root/.openclaw/workspace/                  # 外部工作区
├── hr-copilot-os/                          # 小五的项目
├── [其他外部项目]/                         # 其他团队的项目
└── memory/                                 # 外部项目记忆
```

**规则**：
- ❌ **禁止修改** `workspace/` 下的任何文件
- ❌ **禁止部署**外部项目
- ❌ **禁止读取**敏感文档（除非老板授权）

---

## 🚫 禁止行为（红线）

| 行为 | 正确做法 | 错误做法 |
|------|---------|---------|
| 修改代码 | 只改 `workspace-manager/` | ❌ 修改 `workspace/` |
| 修改样式 | 只改团队项目 | ❌ 修改 hr-copilot-os/ |
| 部署项目 | 只部署团队项目 | ❌ 部署外部项目 |
| 读取文档 | 只读团队规范 | ❌ 读取外部项目文档 |

---

## ✅ 正确流程

### 行动前检查（强制）

```
□ 这个文件/项目在 /root/.openclaw/workspace-manager/ 下吗？
□ 这是祖龙团队的项目吗？
□ 我是否有权修改这个项目？
```

**任一答案为「否」→ 停止 → 请示老板**
