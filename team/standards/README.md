## 1. 视觉设计规范

### 1.1 核心理念

**年轻化 · 科技感 · 色彩丰富 · 现代化**

**参考对象**: 飞书、Ant Design 5.0、字节系产品

### 1.2 色彩体系

#### 主色

| 用途 | HEX/渐变 | 说明 |
|------|----------|------|
| 主色 | `#3370FF` | 飞书蓝，所有主按钮、关键交互 |
| 主色渐变 | `linear-gradient(135deg, #3370FF, #722ED1)` | 按钮背景、卡片装饰 |
| 主色悬停 | `#2860E1` | 按钮悬停态 |
| 主色点击 | `#1D4FB8` | 按钮点击态 |

#### 辅色

| 色系 | 色值 | 用途 |
|------|------|------|
| 科技紫 | `#722ED1` → `#9254DE` | 渐变过渡、强调区域 |
| 清新青 | `#13C2C2` → `#36CFC9` | 成功状态、数据可视化 |
| 深邃蓝 | `#096DD9` → `#1890FF` | 次级主色、链接色 |
| 活力紫 | `#B37FEB` → `#D3ADF7` | 背景装饰、卡片点缀 |

#### 强调色

| 用途 | 色值 | 场景 |
|------|------|------|
| 活力橙 | `#FA8C16` | 警告、重要提示、CTA 按钮 |
| 热情粉 | `#F759AB` | 活动运营、年轻态装饰 |
| 荧光青 | `#00D9C0` | 高亮标签、新功能标识 |
| 成功绿 | `#52C41A` | 成功状态、正向反馈 |
| 警示红 | `#FF4D4F` | 错误、删除、危险操作 |

#### 背景色

| 层级 | 色值/渐变 | 说明 |
|------|-----------|------|
| 主背景 | `linear-gradient(180deg, #F5F7FF 0%, #EBF1FF 100%)` | **所有页面背景** |
| 次级背景 | `#F0F2F5` → `#E8ECF1` | 内容区分隔 |
| 卡片背景 | `rgba(255, 255, 255, 0.95)` + 玻璃态 | **所有卡片** |
| 深色背景 | `linear-gradient(135deg, #1D2129 0%, #2E3340 100%)` | 夜间模式 |

#### 中性色

| 用途 | 色值 |
|------|------|
| 主文字 | `#1D2129` |
| 次级文字 | `#4E5969` |
| 辅助文字 | `#86909C` |
| 占位文字 | `#C9CDD4` |
| 边框 | `#E5E6EB` |
| 分割线 | `#F2F3F5` |

### 1.3 组件样式

#### 卡片组件

```css
.card {
  border-radius: 12px;
  background: rgba(255, 255, 255, 0.95);
  backdrop-filter: blur(10px);
  box-shadow: 0 4px 16px rgba(51, 112, 255, 0.12);
  border: 1px solid rgba(51, 112, 255, 0.08);
  transition: all 0.3s ease-in-out;
}

.card:hover {
  transform: translateY(-2px);
  box-shadow: 0 8px 24px rgba(51, 112, 255, 0.18);
}
```

**关键点**:
- ✅ 圆角：**12px**（统一）
- ✅ 玻璃态：`backdrop-filter: blur(10px)`
- ✅ 阴影：蓝色阴影，非灰色
- ✅ 悬停：上浮 2px + 阴影加深

#### 按钮组件

```css
.btn-primary {
  background: linear-gradient(135deg, #3370FF, #5C7CFA);
  border-radius: 8px;
  padding: 10px 24px;
  font-weight: 600;
  color: #FFFFFF;
  border: none;
  cursor: pointer;
  transition: all 0.25s ease-in-out;
}

.btn-primary:hover {
  transform: translateY(-2px);
  box-shadow: 0 6px 20px rgba(51, 112, 255, 0.35);
}
```

**关键点**:
- ✅ 主按钮：**渐变背景**（不是纯色）
- ✅ 圆角：**8px**（统一）
- ✅ 悬停：**上浮 2px** + 阴影
- ✅ 字重：**600**（加粗）

#### 输入框

```css
.input {
  border-radius: 8px;
  border: 2px solid #E5E6EB;
  background: #FFFFFF;
  padding: 10px 14px;
  font-size: 14px;
  transition: all 0.2s ease;
}

.input:focus {
  border-color: #3370FF;
  box-shadow: 0 0 0 3px rgba(51, 112, 255, 0.12);
  outline: none;
}
```

**关键点**:
- ✅ 圆角：**8px**（统一）
- ✅ 边框：**2px**（不是 1px）
- ✅ 聚焦：**蓝色光晕**（box-shadow）

### 1.4 动效规范

#### 过渡时间

| 场景 | 时长 | 缓动函数 |
|------|------|----------|
| 微小交互 (悬停) | 150ms - 200ms | `ease-out` |
| 标准过渡 (按钮/输入框) | 200ms - 250ms | `ease-in-out` |
| 大型动效 (卡片/模态框) | 300ms - 350ms | `cubic-bezier(0.4, 0, 0.2, 1)` |
| 页面切换 | 400ms - 500ms | `cubic-bezier(0.65, 0, 0.35, 1)` |

#### 悬停效果（统一）

```css
:hover {
  transform: translateY(-2px);
  transition: all 0.3s ease;
}
```

### 1.5 布局规范

#### 间距系统

| 类型 | 数值 | 使用场景 |
|------|------|----------|
| 微型间距 | 4px / 8px | 图标与文字、标签内边距 |
| 小型间距 | 12px / 16px | 表单元素、按钮内边距 |
| 标准间距 | 20px / 24px | **卡片内边距**、模块间距 |
| 大型间距 | 32px / 40px | 页面分区、大模块间隔 |
| 超大型间距 | 48px / 64px | 页面级留白、Hero 区域 |

#### 卡片布局

- **卡片间距**: 24px (卡片之间)
- **卡片内边距**: 20px - 24px
- **圆角**: 8px (小组件) / 12px (大卡片)
- **最大宽度**: 1200px (内容区域)

## 2. 组件开发规范

### 2.1 核心原则

**前端组件化，后端框架化**

**组件和方法的最大化引用**，禁止每个页面单独写一套组件！

### 2.2 组件库结构

```
frontend/src/components/ui/          # 公共 UI 组件库
├── Button/
│   ├── Button.jsx                  # 按钮组件
│   ├── Button.css                  # 按钮样式
│   └── Button.mdx                  # 组件文档
├── Card/
├── Input/
├── Textarea/
├── Table/
├── List/
├── Select/
├── Checkbox/
├── Radio/
├── Modal/
├── Notification/
├── Loading/
├── Skeleton/
├── Breadcrumb/
├── Pagination/
└── index.js                        # 统一导出
```

### 2.3 正确使用方式

```jsx
// ✅ 正确：引用公共组件
import { Button, Card, Input, Table } from '@/components/ui'

function MyPage() {
  return (
    <Card>
      <Input placeholder="请输入..." />
      <Button type="primary">提交</Button>
      <Table data={data} />
    </Card>
  )
}

// ❌ 错误（禁止）：页面内联样式
function MyPage() {
  return (
    <div className="card" style={{...}}>
      <input className="input" style={{...}} />
      <button className="btn" style={{...}}>提交</button>
    </div>
  )
}
```

### 2.4 组件实现要求

#### 样式来源

**100% 还原青鸾 Demo**（`demo.html`）

**禁止行为**:
- ❌ 自己创造样式
- ❌ 修改色值
- ❌ 修改圆角
- ❌ 修改间距
- ❌ "差不多就行"

**正确做法**:
- ✅ 直接复制 Demo 的 CSS
- ✅ 逐像素对比 Demo 页面
- ✅ 用取色器验证色值
- ✅ 用开发者工具验证间距

#### 组件文档

每个组件必须包含 `.mdx` 文档：

```markdown
# Button 按钮

## 基础用法

```jsx
<Button type="primary">主按钮</Button>
```

## API

| 参数 | 说明 | 类型 | 默认值 |
|------|------|------|--------|
| type | 按钮类型 | `'primary' \| 'default' \| 'text'` | `'default'` |
| size | 按钮大小 | `'large' \| 'middle' \| 'small'` | `'middle'` |
| disabled | 禁用状态 | `boolean` | `false` |

## 样式规范

- 主按钮：渐变背景 `linear-gradient(135deg, #3370FF, #722ED1)`
- 圆角：8px
- 悬停：上浮 2px + 阴影加深
```

### 2.5 违规处理

| 行为 | 正确做法 | 违规处理 |
|------|---------|---------|
| 页面内联样式 | 使用公共组件 | ❌ 打回重写 |
| 自己创造样式 | 提醒产品先定义 | ❌ 打回重写 |
| 不引用组件库 | 统一引用 `@/components/ui` | ❌ 打回重写 |
| 重复代码 | 提取为公共方法 | ❌ 打回重构 |

## 3. 项目边界规范

### 3.1 工作区边界

#### ✅ 祖龙团队工作区（可操作）

```
/root/.openclaw/workspace-manager/          # 祖龙团队工作区
├── team/                                   # 团队管理文档
├── projects/                               # 团队项目（可以有多个）
│   ├── qingluan-org/                       # 青鸾组织架构系统

```
/root/.openclaw/workspace/                  # 外部工作区
├── hr-copilot-os/                          # 小五的项目
├── [其他外部项目]/                         # 其他团队的项目
└── memory/                                 # 外部项目记忆
```

**规则**:
- ❌ **禁止修改** `workspace/` 下的任何文件
- ❌ **禁止部署**外部项目
- ❌ **禁止读取**敏感文档（除非老板授权）

### 3.2 行动前检查（强制）

```
□ 这个文件/项目在 /root/.openclaw/workspace-manager/ 下吗？

□ 我是否有权修改这个项目？

**任一答案为「否」→ 停止 → 请示老板**

## 4. 研发流程规范

### 4.1 核心原则

**不预设周期，做完即汇报**

### 4.2 严禁行为

- ❌ "预计 XX 点完成"
- ❌ "排期 X 人天"
- ❌ "17:00 完成 A，18:00 完成 B"

### 4.3 正确做法

- ✅ "立即执行，完成后立即汇报"
- ✅ 完成后直接说"已完成，请验收"

### 4.4 待办清单机制

**每个成员必须在自己记忆文件中维护待办清单**：

```markdown
## 📋 待办清单

| 状态 | 任务 | 优先级 | 备注 |
|------|------|--------|------|
| 🔥 进行中 | XXX | P0 | ... |
| ⏳ 即将做 | XXX | P1 | ... |
| 📝 未开始 | XXX | P2 | ... |
| ✅ 已完成 | XXX | - | 待汇报 |
```

**要求**:
- 动态更新任务状态
- 任务变更时立即更新清单
- 祖龙定期检查各成员清单

## 5. 协作机制

### 5.1 产品 - 研发协作流程

```
1. 青鸾定义组件规范（Demo 页面）
   ↓
2. 麒麟按 Demo 实现公共组件
   ↓
3. 页面开发时引用公共组件
   ↓
4. 遇到新组件 → 青鸾补充定义 → 麒麟实现
```

### 5.2 产品职责（青鸾 🐦‍🔥）

- ✅ 全量定义组件规范（Demo 页面）
- ✅ 包含所有状态（正常/悬停/聚焦/禁用）
- ✅ 提供完整代码示例
- ✅ 遇到新需求时先更新 Demo

### 5.3 研发职责（麒麟 🐅）

- ✅ 按产品 Demo 实现公共组件
- ✅ 提取样式到组件文件
- ✅ 确保所有页面统一引用
- ✅ **遇到规范中没有的组件 → 提醒产品先定义**

### 5.4 测试职责（白泽 🦄）

- ✅ 学习视觉规范
- ✅ 创建 UI 检查清单
- ✅ 对照 Demo 逐一验收
- ✅ 输出测试报告

### 5.5 运维职责（重明 🦜）

- ✅ 容器编排
- ✅ 监控部署
- ✅ 安全防御
- ✅ 仅凭《发布许可证书》执行部署
